## Issues

### ISSUE-001: Local agent.js crash — 缺少 institutional-renderer.js

| 欄位 | 值 |
| --- | --- |
| **分類** | ERR |
| **嚴重度** | P0-Critical |
| **受影響** | market-digest（local only）|
| **環境** | local |

**描述**

Local 的 agent.js 在第 170 行 `require('./institutional-renderer')` 引用了一個不存在的模組，導致任何指令都 crash。VPS 上 `institutional-renderer.js` 存在（正常運作），但 local 缺少此檔案，造成本地開發完全無法測試 agent.js。

**重現步驟**

```bash
# Local
cd src/agents/market-digest
node agent.js help
# 實際：
# Error: Cannot find module './institutional-renderer'
# Exit code: 1

# VPS（正常）
node agent.js help
# 實際：輸出 help 訊息，Exit code: 0
```

**根因分析**

`institutional-renderer.js` 存在於 VPS 但未同步到 local git repo。由於 VPS 和 GitHub 是 unrelated histories，VPS 上的新增檔案若未手動同步到 GitHub，local 就會缺少。

**建議修復**

從 VPS 複製 `institutional-renderer.js` 到 local，再 commit 推送。

### ISSUE-002: Local 與 VPS 版本不同步（institutional-renderer.js 未入 repo）

| 欄位 | 值 |
| --- | --- |
| **分類** | STR |
| **嚴重度** | P1-High |
| **受影響** | market-digest |
| **環境** | local / VPS |

**描述**

VPS 有 `institutional-renderer.js` 但 local（GitHub）沒有。這代表 VPS 有直接新增的檔案未透過 GitHub 同步，造成兩個環境的程式碼不一致。

**重現步驟**

```bash
# Local
ls src/agents/market-digest/institutional-renderer.js
# 結果：NOT FOUND

# VPS
ls /home/clawbot/clawd/agents/market-digest/institutional-renderer.js
# 結果：EXISTS
```

**根因分析**

VPS 和 GitHub 是 unrelated histories，VPS 上直接新增的檔案若未透過逐檔同步流程（`git show origin/main:src/... > agents/...`）推送，就會只存在 VPS。此模式容易造成 drift。

**建議修復**

建立清單，定期確認 VPS agents/ 下的所有檔案都有對應到 GitHub，並在 VPS 新增檔案後立刻同步。

### ISSUE-003: `node index.js help` 回傳 exit code 1（Local 和 VPS 均存在）

| 欄位 | 值 |
| --- | --- |
| **分類** | CLI |
| **嚴重度** | P2-Medium |
| **受影響** | market-digest（index.js）|
| **環境** | local / VPS |

**描述**

`node index.js help` 成功顯示使用說明，但 exit code 為 1（應為 0）。Local 和 VPS 均重現此問題。

**重現步驟**

```bash
# Local 和 VPS 相同行為
cd [market-digest 目錄]
node index.js help >/dev/null 2>&1; echo $?
# 預期：0
# 實際：1

node index.js pipeline --help >/dev/null 2>&1; echo $?
# 實際：0（正確）
```

**建議修復**

追蹤 index.js 的 help 指令執行路徑，確保成功時 `process.exitCode = 0`。

### ISSUE-004: `node index.js unknown-cmd` 回傳 exit code 0（VPS 發現）

| 欄位 | 值 |
| --- | --- |
| **分類** | CLI |
| **嚴重度** | P2-Medium |
| **受影響** | market-digest（index.js）|
| **環境** | local / VPS |

**描述**

執行不存在的指令時，exit code 應為非 0，但 VPS 上回傳 0。Local 上 exit code 為 1（正確），但顯示環境變數清單而非「指令不存在」的明確錯誤訊息，DX 不佳。

**重現步驟**

```bash
# VPS
node index.js unknown-cmd 2>&1 | tail -3; echo "EXIT: $?"
# 實際：顯示環境變數清單，EXIT: 0（應為 1）
```

**建議修復**

在 index.js 的 fallback 路徑加入 `process.exit(1)` 和明確的「未知指令」錯誤訊息。
