## Issues

### ISSUE-001: 4 個 agents 缺少 agent.js（主入口不符合 agent-template 規範）

| 欄位 | 值 |
| --- | --- |
| **分類** | STR |
| **嚴重度** | P1-High |
| **受影響** | deploy-monitor、kanban-dashboard、knowledge-digest、optimization-advisor |
| **環境** | local |

**描述**

以下 4 個 agents 完全缺少 `agent.js`，不符合 agent-template 強制規範：

```
deploy-monitor/       → 只有 scripts/ 和 README.md
kanban-dashboard/     → 有 package.json + server/ + client/，但無 agent.js
knowledge-digest/     → 只有 scripts/ 和 README.md
optimization-advisor/ → 只有 scripts/ + data/
```

`security-patrol` 有 `patrol.js` 但非 `agent.js`，功能完整但入口名稱不合規。

**重現步驟**

```bash
cd src/agents
for a in deploy-monitor kanban-dashboard knowledge-digest optimization-advisor; do
  ls $a/agent.js 2>/dev/null || echo "$a: agent.js MISSING"
done
# 4 個全部輸出 MISSING
```

**根因分析**

這些 agents 在 agent-template 規範確立前建立，或只完成骨架（scripts/ 存在但未建立標準入口）。kanban-dashboard 有完整的 Node.js 應用，透過 PM2 啟動 `server/index.js`，未建立統一的 agent.js 包裝層。

**建議修復**

為每個 agent 建立 `agent.js`，至少實作 `help` 和 `status` 指令，代理到實際執行入口。

### ISSUE-002: 5 個 agents 缺少符合 agent-template schema 的根目錄 config.json

| 欄位 | 值 |
| --- | --- |
| **分類** | CFG |
| **嚴重度** | P1-High |
| **受影響** | deploy-monitor、kanban-dashboard、knowledge-digest、optimization-advisor、security-patrol |
| **環境** | local |

**描述**

agent-template 要求根目錄有 `config.json` 並包含 `name`、`version`、`paths.src`、`paths.runtime`：

| Agent | 狀況 |
| --- | --- |
| deploy-monitor | config.json 在 `scripts/` 下（非根目錄），缺 name/version |
| kanban-dashboard | 無 config.json（有 package.json）|
| knowledge-digest | 無 config.json |
| optimization-advisor | config.json 在 `scripts/` 下（非根目錄），缺 name/version |
| security-patrol | config.json 在根目錄，但缺 name/version/paths |

**重現步驟**

```bash
cd src/agents
for a in deploy-monitor kanban-dashboard knowledge-digest optimization-advisor security-patrol; do
  node -e "try{const c=require('./$a/config.json'); \
    console.log('$a name:', c.name||'MISSING')}catch(e){console.log('$a: MISSING')}" 2>/dev/null
done
# 全部缺失
```

**建議修復**

為每個 agent 在根目錄建立包含必要欄位的 `config.json`。

### ISSUE-004: social-digest agent.js 未知指令回傳 exit code 0（應為 1）

| 欄位 | 值 |
| --- | --- |
| **分類** | CLI |
| **嚴重度** | P2-Medium |
| **受影響** | social-digest |
| **環境** | local |

**描述**

`node agent.js <unknown>` 顯示 help 文字後回傳 exit code 0，應回傳 1。

**重現步驟**

```bash
cd src/agents/social-digest
node agent.js unknown >/dev/null 2>&1; echo $?
# 預期：1
# 實際：0
```

**建議修復**

在 agent.js 的 default/unknown 分支加入 `process.exit(1)`。

### ISSUE-005: security-patrol config.json 的 telegram.chat_id 為無效佔位符

| 欄位 | 值 |
| --- | --- |
| **分類** | DX |
| **嚴重度** | P2-Medium |
| **受影響** | security-patrol |
| **環境** | local |

**描述**

`config.json` 的 `telegram.chat_id` 值為 `"REDACTED_CHAT_ID"` 字串佔位符，而非 `"${TELEGRAM_CHAT_ID}"`。`patrol-wrapper.sh` 實際從環境變數讀取（正確），但 config.json 的無效佔位符容易誤導維護者。

**重現步驟**

```bash
grep chat_id src/agents/security-patrol/config.json
# 輸出："chat_id": "REDACTED_CHAT_ID"
```

**建議修復**

改為 `"chat_id": "${TELEGRAM_CHAT_ID}"`，與其他 agents 慣例一致。
