# Daily Title Optimization 每日標題優化 Agent

## 📋 概述

Daily Title Optimization agent 每天凌晨 3 點分析知識庫和任務標題，使用 AI 優化標題的清晰度和可搜尋性。

## 🎯 目的

- **標題改善**：使用 AI 重寫模糊的標題
- **關鍵字提取**：自動加入相關標籤
- **一致性檢查**：統一命名規範
- **可搜尋性提升**：優化標題以提升搜尋命中率

## ⚙️ 設定

```yaml
名稱: daily-title-optimization
排程: "0 3 * * *"  # 每天凌晨 03:00
逾時: 180000       # 3 分鐘
記憶體限制: 80M
```

## 📊 優化範例

```
優化前後對比：

❌ 原標題：「修 bug」
✅ 優化後：「修復 Dashboard 輪詢導致的記憶體洩漏」

❌ 原標題：「研究一下」
✅ 優化後：「研究 React useMemo 在大型列表中的效能影響」

❌ 原標題：「重要!!」
✅ 優化後：「[緊急] 修復 VPS 記憶體不足問題」

規則：
  ✓ 動詞開頭（修復、新增、優化）
  ✓ 包含關鍵元件名稱
  ✓ 簡潔但資訊完整（<80 字元）
  ✓ 標籤一致性（使用既有標籤）
```

## 📝 處理流程

```
1. 載入所有任務和知識條目
2. 識別模糊標題（長度<5 字元或無動詞）
3. 批次送至 Claude Haiku 優化
4. 人工審核模式（儲存建議，不自動套用）
5. 生成優化報告
```

## 📈 效能指標

- **平均執行時間**：1-2 分鐘
- **處理標題數**：每天約 20-30 個
- **Token 消耗**：每天約 10K tokens
