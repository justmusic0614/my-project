### ISSUE-001: agent.js 完全無法載入 — 缺少 institutional-renderer 模組

| 欄位 | 值 |
| --- | --- |
| **分類** | ERR |
| **嚴重度** | P0-Critical |
| **受影響** | market-digest |
| **環境** | local / VPS |

**描述**

agent.js 在第 170 行 `require('./institutional-renderer')` 引用了一個不存在的模組。無論執行 `help`、`status` 或任何指令，agent.js 都會 crash，完全不可用。

**重現步驟**

```bash
cd src/agents/market-digest
node agent.js help
# 預期：顯示 help 訊息
# 實際：
# Error: Cannot find module './institutional-renderer'
# Require stack:
# - .../src/agents/market-digest/agent.js
# Exit code: 1
```

**根因分析**

agent.js:170 的 `require('./institutional-renderer')` 在模組載入階段就執行（頂層 require），即使 help 不需要 renderReport 也會 crash。`renderers/` 目錄下有 `daily-renderer.js`、`tactical-renderer.js`、`weekly-renderer.js`，但沒有 `institutional-renderer.js`。可能是重構後遺留的引用。

**建議修復**

找到正確的 renderer（可能是 `renderers/daily-renderer.js`），修正 require 路徑。或改為 lazy require（在 generateReportFromRuntime 內部才 require）。

### ISSUE-002: config.json 缺少 `name` 欄位，且缺少 `paths.src` 和 `paths.runtime`

| 欄位 | 值 |
| --- | --- |
| **分類** | CFG |
| **嚴重度** | P1-High |
| **受影響** | market-digest |
| **環境** | local / VPS |

**描述**

config.json 不符合 agent-template 的 schema 基準。缺少 `name` 欄位，`paths` 物件中沒有 `src` 和 `runtime` 路徑。

**重現步驟**

```bash
cd src/agents/market-digest
node -e "const c = require('./config.json'); \
  console.log('name:', c.name || 'MISSING'); \
  console.log('paths.src:', c.paths?.src || 'MISSING'); \
  console.log('paths.runtime:', c.paths?.runtime || 'MISSING');"
# 實際輸出：
# name: MISSING
# paths.src: MISSING
# paths.runtime: MISSING
```

**根因分析**

market-digest 的 config.json 是早期設計（v2.0），未遵循後來建立的 agent-template 標準。`paths` 物件列了 data、cache、output 等路徑，但缺少 template 要求的 `src` 和 `runtime`。

**建議修復**

在 config.json 中加入 `"name": "market-digest"` 和 `paths.src: "backend"`、`paths.runtime: "data/runtime"`。

### ISSUE-003: agent.js showStatus() 引用不存在的 config schema

| 欄位 | 值 |
| --- | --- |
| **分類** | ERR |
| **嚴重度** | P2-Medium |
| **受影響** | market-digest |
| **環境** | local / VPS |

**描述**

agent.js:144 引用 `config.data_sources.tw_news` 和 `config.data_sources.intl_news`，但 config.json 中的結構是 `dataSources.rss` 和 `dataSources.api`。即使 agent.js 能載入，`/status` 指令也會 crash。

**重現步驟**

```javascript
// agent.js:144
config.data_sources.tw_news.filter(s => s.enabled)
// 但 config.json 使用 dataSources.rss（非 data_sources.tw_news）
// TypeError: Cannot read properties of undefined
```

**根因分析**

agent.js 是更早版本的程式碼，config schema 後來改為 camelCase（dataSources），但 agent.js 未同步更新。

**建議修復**

更新 showStatus() 使用正確的 config 路徑，或評估 agent.js 是否需要重寫以匹配目前 index.js 架構。

### ISSUE-004: `node index.js help` 回傳 exit code 1

| 欄位 | 值 |
| --- | --- |
| **分類** | CLI |
| **嚴重度** | P2-Medium |
| **受影響** | market-digest (index.js) |
| **環境** | local / VPS |

**描述**

`node index.js help` 成功顯示使用說明，但 exit code 為 1。成功的 help 輸出應該回傳 0。

**重現步驟**

```bash
cd src/agents/market-digest
node index.js help >/dev/null 2>&1; echo $?
# 預期：0
# 實際：1

# 對比：
node index.js pipeline --help >/dev/null 2>&1; echo $?
# 實際：0（正確）
```

**根因分析**

index.js 的 help 指令處理路徑可能在某個 catch block 或 fallback 路徑中，導致 process.exitCode 被設為 1。

**建議修復**

追蹤 index.js help 指令的執行路徑，確保成功時 `process.exitCode = 0`。
