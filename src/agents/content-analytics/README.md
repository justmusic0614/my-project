# Content Analytics 內容分析 Agent

## 📋 概述

Content Analytics agent 每天凌晨 4 點分析知識庫內容，生成主題分布、關鍵字趨勢、和內容品質報告。

## 🎯 目的

- **主題分析**：使用 LDA 識別主要主題
- **趨勢追蹤**：分析關鍵字出現頻率變化
- **品質評估**：檢測重複內容和資訊密度
- **視覺化報告**：生成圖表和統計儀表板

## ⚙️ 設定

```yaml
名稱: content-analytics
排程: "0 4 * * *"  # 每天凌晨 04:00
逾時: 300000       # 5 分鐘
記憶體限制: 150M
```

## 📊 分析維度

```
主題分布：
  ├─ 技術相關：45%
  ├─ 產品設計：25%
  ├─ 商業策略：20%
  └─ 其他：10%

關鍵字趨勢（過去 7 天）：
  ↗ React: +15%
  ↗ AI Agent: +30%
  ↘ TypeScript: -5%

內容品質指標：
  ✓ 平均字數：280 字/條目
  ✓ 重複率：<5%
  ✓ 外部連結數：平均 2 個/條目
```

## 🔧 依賴套件

| 套件 | 版本 | 用途 |
|------|------|------|
| natural | ^6.0.0 | NLP 分析 |
| chart.js | ^4.0.0 | 圖表生成 |
| @anthropic-ai/sdk | ^0.32.0 | AI 主題摘要 |

## 📈 效能指標

- **平均執行時間**：2-3 分鐘
- **分析條目數**：全部知識庫（~500 個）
- **記憶體使用**：尖峰 100-120 MB
