# Pipeline 依賴模型

任何 pipeline 架構都遵循這個模式。修改任何一層時，用此圖檢查下游影響。

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  收集    │───▶│  轉發    │───▶│  處理    │───▶│  驗證    │───▶│  渲染    │───▶│  推播    │
│ Collect  │    │ Forward  │    │ Process  │    │ Validate │    │ Render   │    │ Publish  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     │                │               │               │               │               │
 新增資料源      資料欄位傳遞     邏輯/演算法      閾值/規則       顯示格式        推送管道
 API 配置        schema 對齊      AI 分析         config.json     模板更新        通知設定
```

## 連鎖修改檢查規則

- **改「收集」層** → 檢查轉發是否傳遞新欄位 → 處理是否使用 → 驗證是否有規則
- **改「處理」層** → 檢查渲染是否顯示新結果 → 驗證是否有規則
- **改「驗證」層** → 檢查 config.json 閾值是否合理
- **改「配置」** → 檢查所有讀取該配置的模組
