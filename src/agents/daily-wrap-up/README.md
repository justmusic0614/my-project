# Daily Wrap-up 每日總結 Agent

## 📋 概述

Daily Wrap-up agent 每天凌晨 1 點生成前一天的工作總結，包含任務完成狀況、知識摘要、和明日建議。

## 🎯 目的

- **任務總結**：統計完成/進行中/待辦任務數量
- **知識回顧**：整合當日所有知識條目
- **進度分析**：計算完成率和效率指標
- **明日規劃**：根據優先度建議明日重點

## ⚙️ 設定

```yaml
名稱: daily-wrap-up
排程: "0 1 * * *"  # 每天凌晨 01:00
逾時: 120000       # 2 分鐘
記憶體限制: 80M
```

## 📊 生成內容

```
資料來源：
  ├─ Kanban tasks.json（任務資料）
  ├─ knowledge-digest（知識條目）
  ├─ Agent 執行日誌
  └─ notification 歷史

報告結構：
  ┌─ 📊 今日統計
  │  ├─ 完成任務：X 個
  │  ├─ 進行中：Y 個
  │  └─ 完成率：Z%
  │
  ├─ 📝 知識摘要
  │  └─ 今日新增 N 個知識條目
  │
  ├─ 🎯 重點成就
  │  └─ AI 挑選最重要的 3 件事
  │
  └─ 💡 明日建議
     └─ 根據優先度排序的待辦清單
```

## 🔧 依賴套件

| 套件 | 版本 | 用途 |
|------|------|------|
| @anthropic-ai/sdk | ^0.32.0 | AI 分析 |
| date-fns | ^4.1.0 | 日期處理 |

## 📝 範例輸出

```markdown
# 每日總結 - 2026-02-13

## 📊 今日統計
- ✅ 完成任務：5 個
- 🔄 進行中：3 個
- ⏸️ 待辦：12 個
- 📈 完成率：71%

## 📝 知識摘要
今日新增 8 個知識條目，涵蓋：
- React 效能優化技巧（3 則）
- AI Agent 設計模式（2 則）
- VPS 部署經驗（3 則）

## 🎯 重點成就
1. 完成 Kanban Dashboard 優化
2. 修復 Calendar 重疊問題
3. 新增 Agent 規格查看功能

## 💡 明日建議
1. [高優先] 部署 Dashboard 至 VPS
2. [中優先] 編寫其他 Agent 的 README
3. [低優先] 優化輪詢頻率
```

## 📈 效能指標

- **平均執行時間**：30-45 秒
- **處理資料量**：當日所有任務和知識條目
- **記憶體使用**：尖峰 50-70 MB
- **Token 消耗**：每天約 5K tokens

## 🔍 監控

日誌儲存於：`~/.openclaw/logs/daily-wrap-up.log`
